# duanxianxia 项目

长期项目：围绕 `https://duanxianxia.com/` 建立可复用的数据抓取、结构化输出、以及每次抓取后的持久化存储能力。

## 项目目标

1. 对用户点名的页面/榜单/表格建立稳定抓取器。
2. **每次抓取都要落盘存储**，保留时间戳与原始结构化结果。
3. 后续支持：
   - 即时查询
   - 历史回看
   - 多表批量抓取
   - 对比同一表在不同时间的变化

## 当前规则（用户明确要求）

- 这是一个**长期项目**，不是一次性任务。
- 用户要求抓取的表格数据：**每次都要存储**。
- 默认输出应尽量完整，**不得擅自截断**，除非用户明确要求精简。
- 时间展示默认按**北京时间**理解与输出。
- 取数时必须尽量以用户指定的**准确路径和截图字段**为准，避免用“相似页面”顶替。
- 所有用户可见表格默认使用**完整中文表头**；只有用户明确要求“原始字段 / 英文表头 / CSV / JSON 原样”时，才切回英文原字段。
- 后续数据集口径一律按 **`dataset_id + source_path`** 锁定，不再按中文名猜。
- `pool.hot`（热门）与 `rank.hot_stock_day` / `hot_stock_day`（热度榜（日））必须严格区分。
- `review.ltgd.range` 的正式展示固定为：**4 个周期表（5/10/20/50）**，且每个表内按 **主板 / 创业科创板 / 北交所** 展示。
- `review.ltgd.range` 在盘后推送中默认要**列全结果**，不要只给前几条预览；格式要尽量简洁整齐。
- 详细固定约定见：`docs/user-contract.md`
- 后续接续优先入口：`docs/project-handbook-current.md`
- 时段分组固定为：盘前（7/8/9/10/16/17）、盘中（1/4/2/3/17）、盘后（6/5）
- 盘前固定：工作日北京时间 `09:25`，随机延迟 `5-15 秒` 自动抓取并保存（当前包含竞价4表 + qxlive主标签汇总表 + 主页 qxlive 顶部指标按钮组）
- 盘后固定：工作日北京时间 `17:20`，随机延迟 `0-5 分钟` 自动抓取并保存
- 盘后自动推送：除了回执摘要外，webhook / announce 文案必须带盘后明细数据，不能只给“成功/失败/报告文件”
- 盘中固定：不自动抓；由用户盘中随时下指令后抓取并保存（当前包含盘中4表 + 主页 qxlive 顶部指标按钮组）
- 个股资金流向（前100名）固定：每个交易日自动抓取并保存 **两次**：
  - **10:01（延迟 0-45 秒）** 抓一次，并与**盘中 4 张表**一起抓
  - **盘后** 与盘后数据一起再抓一次
- 后续若无用户明确确认：**不得随意改抓取源、落盘结构、正式输出字段、字段顺序、表拆分方式、时段分组与执行时点**。

## 目录说明

- `captures/`

- `config/`

- `docs/`
  - 字段说明、路径说明、项目设计说明
